WHAT THE SYSTEM MEASURES OBSERVABLE PROXIES · DISCLOSED LIMITS

κ   ε   Ic   Φ*

Four readings. None is a verdict by itself. Together they describe how a structure is carrying meaning through change.

  1. 01 κ

    Stress

    Pressure from load, retries, bottlenecks, and structures carrying too much work.

    How tightly is the system being forced to operate?
  2. 02 ε

    Entropy leak

    Observed drift, unresolved contradiction, and information losing its supporting path.

    How much meaning is escaping its original structure?
  3. 03 Ic

    Information capacity

    A local estimate of what the current graph can carry without structural collapse.

    How much can this structure hold right now?
  4. 04 Φ*

    Coherence

    Whether claims, evidence, constraints, plans, and outcomes still form a usable whole.

    Does the system still make sense as one object?

WHY BOUNDARIES MATTER METHOD NOTE 02

A system is not coherent because its center agrees with itself.

Coherence becomes meaningful at the boundary: where a claim meets evidence, where memory becomes action, where one agent hands state to another. Dynamic Sentience Maps makes those crossings visible before the system turns a private mistake into a public decision.

OPEN ARCHITECTURE BRIDGE API · v0.1 CONTRACT

Open enough to inspect.
Bounded enough to keep.

The public layer stores graph structure, renders the instrument, and exposes observable estimators. A private kernel can return metrics and interventions through one declared bridge. The graph receives the result; the kernel keeps its internals.

01 / OPEN LAYER

What the public can inspect

  • Graph nodes, links, and provenance
  • Telemetry event structure
  • Open metric estimators
  • Intervention display and export
02 / DECLARED CROSSING

{

Returns metrics, interventions, and a kernel version. No private method is implied by the public demo.
