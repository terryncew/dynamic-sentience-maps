# Claim boundary

Dynamic Sentience Maps is an open-core telemetry and graph-processing prototype. The public instrument demonstrates how a structured graph can make pressure, drift, contradictions, boundaries, and evidence paths inspectable.

## The release claims

- The public site renders a deterministic demonstration graph.
- Its three observations change declared positions, readings, stress values, and evidence state.
- Every visible relationship in the graph maps to an explicit source/target/relation record.
- Users can inspect nodes and regions with pointer or keyboard input.
- All Python sources parse and compile.
- Under the declared Python dependencies, the FastAPI application imports and completes an isolated graph, telemetry, metric, and Bridge API runtime smoke test.
- The repaired backend retains the repository's declared API route contract.

## The release does not claim

- That the demonstration readings are scientifically validated universal metrics.
- That `κ`, `ε`, `I_c`, or `Φ*` proves consciousness, sentience, truth, safety, or system health.
- That the sample graph describes a live production system.
- That a private bridge implementation has been independently verified; the runtime smoke covers only the open demonstration kernel.
- That a content hash or signature is sufficient evidence on its own.

The numbers in `docs/demo-data.js` are disclosed interface fixtures. A receiver must evaluate provenance, freshness, evidence, and policy before using results from any real deployment.
